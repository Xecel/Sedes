## Commit Message Guidelines## Format:## <type>(<scope>): <short description>## <optional longer description>## Types:## - feat:     A new feature## - fix:      A bug fix## - docs:     Documentation changes## - style:    Code style changes## - refactor: Code refactoring## - test:     Adding or modifying tests## - chore:    Routine tasks, maintenance, or tooling changes## Scope:## - Specify the module, component, or area affected by the change## Examples:## - feat(user-auth): add password reset functionality## - fix(bug): resolve issue with login form## - docs(readme): update installation instructions
